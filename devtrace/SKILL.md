# DevTrace

将原始开发记录整理成可复盘、可追溯、可导出的结构化结果。优先忠实反映原始记录，不补造事实，不把闲聊误判为主线。

## 处理目标

默认产出以下内容：

- 会话主题
- 总体总结
- 主线脉络
- 关键节点
- 核心内容
- 原文证据映射
- 按需导出的 Markdown/TXT

## 输入范围

优先处理以下输入：

- `.md` 聊天记录
- `.txt` 聊天记录
- 需求推进记录
- 开发讨论原文
- 调试过程复盘材料
- 多段按时间排列的项目过程文本

如果用户给的是多份材料，先按时间和语义关系整理，再统一总结。

## 工作原则

始终遵循以下规则：

1. 保留原始语义，不凭空补充背景、决策或结果。
2. 优先恢复“推进过程”，不要只做松散摘要。
3. 总结尽量能回溯到原文片段、消息区间或段落证据。
4. 弱化零碎细节，突出真正推动过程前进的节点。
5. 如果证据不足，明确写“待确认”或“证据不足”。

## 快速决策

### 何时直接执行

- 整理开发聊天记录
- 梳理需求推进过程
- 提炼调试主线
- 从讨论文本中抽关键节点
- 生成可复盘总结
- 生成 Markdown/TXT 摘要

直接使用本 skill。

## 标准工作流

按以下顺序执行：

### 1. 读取输入

确认输入类型、长度、可读性和边界质量。

优先识别：

- 角色
- 时间顺序
- 消息边界
- 问题、决策、行动、结果相关片段

### 2. 划分消息或语义块

优先把**同一条长消息**视为一个完整语义单元。不要因为消息内部有标题、代码块、表格就拆成多个关键步骤。

如果原文没有清晰消息边界，则按语义分段。

### 3. 提炼会话主题

用一句话概括这份记录主要在推进什么事情。主题应聚焦主任务，不要空泛。

### 4. 生成总体总结

先回答：

- 起点是什么
- 过程中经历了什么
- 为什么发生变化
- 当前走到哪里
- 是否有明确结论或遗留项

### 5. 还原主线脉络

按阶段或关键推进点组织主线。优先保留：

- 问题发现
- 根因诊断
- 方案尝试
- 失败转折
- 策略升级
- 最终选择

关键路径通常控制在 **4~7 步**。每一步尽量对应一轮完整讨论，而不是一句话一个点。

### 6. 抽取关键节点

优先识别：

- 明确提出需求或目标
- 暴露关键问题或阻塞
- 方案比较与取舍
- 方向切换
- 失败后的升级诊断
- 结果确认
- 遗留问题或后续动作

### 7. 提炼核心内容

抽取值得长期保留的信息，例如：

- 关键决策
- 关键结论
- 关键实现思路
- 关键风险或限制
- 后续待办

如果多条相邻内容本质上在讲同一个问题，合并处理；如果某一步只是前一步补充说明，尽量并入前一步。

### 8. 绑定原文证据

对重要结论尽量附带原文依据。优先使用：

- 原文片段摘要
- 段落范围
- 消息区间
- 锚点消息

不要长篇复制原文；优先“摘要 + 定位”。

### 9. 生成结果

将结果整理成稳定结构，保证后续可复盘、可导出、可继续编辑。

### 10. 按需导出

如果用户要求导出：

- Markdown：保留完整标题层级
- TXT：保留主要结构并适度压缩

## 输出结构

默认按以下结构输出。

### 1. 会话主题

一句话概括主任务。

### 2. 总体总结

用 1~3 段说明背景、推进过程、当前结果。

### 3. 主线脉络

按时间或阶段列出主要推进路径。优先体现“为什么从上一步走到下一步”。

### 4. 关键节点

逐条列出重要事件、转折和确认结论。

### 5. 核心内容

整理高价值信息点，适合长期保留。

### 6. 原文证据

为关键结论附尽可能清晰的原文依据或定位信息。

### 7. 导出结果

如果用户要求导出，则生成 Markdown 或 TXT 版本。

## 关键判断规则

### 应判定为关键节点的内容

- 需求明确
- 问题暴露
- 方案比较
- 方向切换
- 失败升级
- 基线回退
- 新方案收敛
- 完成确认
- 遗留待办

### 不应过度提炼的内容

- 寒暄
- 无信息增量的重复确认
- 与主线无关的闲聊
- 不改变主线的局部修修补补

### 特别关注的链条

当记录里出现以下模式时，优先还原完整链条：

- 失败 -> 升级诊断 -> 新方案
- 补丁未生效 -> 最小化验证 -> 基线回退
- 发现原方案不成立 -> 调整评估标准 -> 改用新路径

## 长文本策略

当输入很长时：

1. 先粗分阶段
2. 再提炼每阶段摘要
3. 最后汇总全局主线
4. 再补关键节点与证据

优先保证主线清楚，再补细节。

## 输出风格

- 使用中文
- 先结论后细节
- 分点清晰
- 避免宣传式表述
- 不夸大结论
- 信息不足时明确说明限制
