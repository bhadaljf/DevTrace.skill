# Index / Current

DevTrace 的入口层有两个核心文件：

- `01_CURRENT.md`
- `00_INDEX.md`

它们的共同特点是：

- 都是入口视图
- 都不是事实对象本体
- 都不能替代 `TraceUnit`

## `01_CURRENT.md`

`01_CURRENT.md` 是当前状态入口。

它回答：

- 当前主线
- 当前问题
- 当前 blocker
- 下一步
- 最近变化

它的作用是：

- 让 AI 快速进入“现在”
- 给继续工作提供最短路径

## `00_INDEX.md`

`00_INDEX.md` 是生成内容索引。

它回答：

- 最近生成了哪些内容
- 最近有哪些内容值得回看
- 时间线上最近记录了什么
- 哪些 tag 与记录类型值得继续读

它记录的主要内容包括：

- 当前会话总结
- 阶段总结
- review
- 日报
- 周报
- 月报
- 定时生成的记录内容

它的作用是：

- 让用户快速进入“最近生成了什么”
- 给 AI 后续读取提供导航入口

## 入口层之后怎么走

先读完 current / index 后，不应直接全读所有 unit。

正确顺序是：

1. 读 current
2. 读 index
3. 看 index 中最近生成了什么内容
4. 再根据任务决定是否需要继续读：
   - 相关总结 / review / 日报 / 周报 / 月报
   - 相关 `TraceUnit`

## current 的更新原则

`01_CURRENT.md` 只保留当前仍有效的状态。

更新方式应按：

- 覆盖
- 保留
- 追加

来处理，而不是机械填槽。

## index 的更新原则

`00_INDEX.md` 只在产生新的生成内容时更新。

例如：

- 会话总结生成后
- review 生成后
- 日报 / 周报 / 月报生成后
- 定时任务生成后

单纯新建 unit 或续写 unit，不应默认直接写成 index 主记录项。
