# Output Schema

本文件描述 DevTrace 风格的输出组织方式。需要贴近原项目数据结构或界面分区时读取。

## 核心结果层

优先组织成以下 3 层：

### 1. Summary

对应整体总结区域，通常包括：

- `title`：本次分析标题
- `overview`：总体总结
- `key_points`：重点内容
- `risks`：风险
- `next_steps`：后续动作

### 2. Flow

对应主线流程区域，通常包括：

- `headline`：流程标题
- `nodes`：阶段节点列表

每个节点可描述：

- 标题
- 简述
- 详细说明
- 锚点消息
- 是否是主线或分支

### 3. Decision / Core Content

对应核心内容卡片区域，通常包括：

- 标题
- 叙述
- 备选项或分支思路
- 锚点消息

## 建议的人类可读输出

如果直接输出给用户，优先按以下顺序：

1. 会话主题
2. 总体总结
3. 主线脉络
4. 关键节点
5. 核心内容
6. 原文证据
7. 后续动作

## 证据字段的表达

证据优先使用以下形式之一：

- 锚点消息 ID
- 证据消息 ID 列表
- 原文片段摘要
- 段落范围

如果没有明确 ID，也可以写：

- “见前半段方案比较处”
- “见失败升级讨论段”
- “见最终确认阶段”

## 导出格式

### Markdown

保留标题层级和分节，适合归档和分享。

### TXT

压缩层级但保留主要结构，适合快速阅读。
