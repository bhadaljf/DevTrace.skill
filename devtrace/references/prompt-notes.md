# Prompt Notes

本文件整理自 DevTrace 的 prompt 实验素材。需要压缩主线、控制步数或突出失败升级链时读取。

## 1. 长消息优先

优先把同一条长消息视为一个完整语义单元，不要因为内部有标题、代码块、表格就拆成多个关键步骤。

## 2. 主线优先

目标不是细拆所有点，而是还原对话主线推进。输出时优先保留：

- 问题发现
- 根因诊断
- 方案尝试
- 失败转折
- 策略升级
- 最终选择

## 3. 步数压缩

关键路径通常控制在 4~7 步。每一步尽量对应一轮完整讨论，而不是一句话一个点。

## 4. 调试复盘视角

如果材料像研发调试过程，重点回答：

- 一开始遇到了什么问题
- 为什么原方案不行
- 中途换了哪些思路
- 每次切换的依据是什么
- 最后收敛到什么方案

## 5. 演示压缩视角

如果用户想做演示或比赛展示：

- 输出少而准的关键步骤
- 合并相邻重复内容
- 将补充说明并入前一步

## 6. 失败升级链条

优先识别：

- 补丁未生效
- 实验失败
- 最小化验证
- 基线回退
- 发现作弊或误判
- 两阶段方案

如果某个节点只是局部修修补补且没有改变主线，不要单独列一步。
