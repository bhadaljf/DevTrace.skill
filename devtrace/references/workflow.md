# DevTrace Workflow

本文件描述 DevTrace 在“开发过程整理”场景下的标准处理顺序。需要最大程度贴近原项目时读取。

## 目标

将原始开发记录转成可复盘的结构化结果，重点不是“最后改了什么代码”，而是：

- 原始聊天记录
- 主线脉络梳理
- 核心内容提炼
- 原文证据跳转
- Markdown / TXT 导出

## 推荐流程

### 1. 导入

支持 `.md` / `.txt`，也可处理其他被转成纯文本的开发记录。

### 2. 解析

先把输入拆成消息或语义块。优先保留长消息的完整语义，不做过细切分。

### 3. 粗分析

先生成粗粒度判断：

- 会话主题
- 总体概览
- 粗阶段划分
- 候选关键区域
- 候选关键路径

### 4. 主线整理

将过程组织成一条可读主线，优先突出真正推动过程前进的转折点，而不是罗列所有细节。

### 5. 重点提炼

提炼：

- 关键点
- 风险
- 后续动作
- 关键决策或取舍

### 6. 证据绑定

对重要结论绑定消息锚点、消息区间或原文片段摘要，支持回溯。

### 7. 导出

根据用户需求整理成 Markdown 或 TXT。

## 使用偏好

在 DevTrace 语境下，通常应：

- 弱化碎片细节
- 突出阶段推进
- 合并相邻重复内容
- 用少而准的关键步骤表达主线
