# Examples

## 示例 1：当前会话沉淀为一个大事件

场景：

- 本轮讨论围绕同一个主题推进
- 中间出现了问题、判断、结论、下一步
- 但它们都属于同一个大事件

处理方式：

- 优先沉淀成一个 `TraceUnit`
- 不要因为出现了 `problem / decision / next-step` 就自动拆成多个 unit

## 示例 2：本轮只是旧事件续写

场景：

- 之前已经有一个关于某个系统设计的 `TraceUnit`
- 这一轮只是继续推进、补证据、更新状态

处理方式：

- 定位旧 `TraceUnit`
- 用 `extend-unit` 续写
- 追加 `details / evidence`
- 需要时更新 `summary / status / relation`
- 如果本轮还生成了新的总结 / review / 日报周报，再更新 `00_INDEX.md`

## 示例 3：一轮里真的出现两个独立大事件

场景：

- 一部分内容在推进系统设计主线
- 另一部分内容单独形成了新的运行 blocker
- 两者未来都值得单独继续追踪

处理方式：

- 可以拆成两个 `TraceUnit`
- 但前提是真正出现了两个独立的大事件
- 不是为了覆盖聊天内容而强行切多块
